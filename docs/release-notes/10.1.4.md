This is a minor release of WebSharper 10.1. NuGet version `10.1.4.673`.

Templates for .NET Core SDK: `dotnet new install WebSharper.Templates@10.1.4.673`

## Enhancements
* #1508 Expanded JS parser to support all valid numeric literals, including binary (e.g. `0b01011001`), octal (e.g. `0o31`), and `BigInt` literals (e.g. `80658175170943878571660636856403766975289505440883277824000000000000n`), and numeric separators (e.g. `1_000_000`)
* #1577 Expanded JS parser to support arrow functions with no parameters, single parameter, parameter list, and either expression or block body. However, some related JS features are still not supported by WebSharper (other than using `JS.Verbatim` which preserves a code snippet as-is), these are: object/array destructuring, default parameters, rest parameters.
