This is a major release of WebSharper 10. NuGet version `10.1.0.667`.

Templates for .NET Core SDK: `dotnet new install WebSharper.Templates@10.1.0.667`

## Enhancements
* #1574 Add `debugger` keyword handling. Use `JS.Debugger` to insert it.
* #1568 Turn on ReadyToRun for compilers for Windows and glibc-based Linux distributions.
  * Measured builds of `websharper-web` template: average time down to 9.9s from 15.6s on Windows.
* #1573 Updated `WebSharper.AspNetCore` for `net5.0+`. No more dependence on deprecated `IHostingEnvironment` interface.
* #1566 Better time measurement and download resources logging.
  * Some steps are separated better, summaries are added for unpack
  * New MSBuild property `WebSharperLogImportance` now sets WebSharper compiler messages to a different importance level than default `Low` (shown for `Detailed`/`Diagnostic` verbosity). For example, add `<WebSharperLogImportance>High</WebSharperLogImportance>` will show the WebSharper logs for all verbosities.
* #1566 To improve unpacking performance, some known non-WebSharper references are fixed. All `Microsoft.*` and `System.*` libraries are treated as a heuristic, do not use these as your WebSharper project names.
* #1559 Support `.mjs` and `.cjs` resources.
* #1565 Translate server-side expressions in F# top-level actions (module-bound `do`) or `fsi` submissions.

## Fixes
* #1558 Fix SPA resource unpacking and `extra.files` handling for `spa` projects. Output location for a `{FilePattern}` is now working as intended: https://docs.websharper.com/core/resources#embedding-files-with-extrafiles-file
