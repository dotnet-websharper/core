This is a major release of WebSharper. NuGet version `10.0.0.651`.

Templates for .NET Core SDK: `dotnet new install WebSharper.Templates@10.0.0.651`

## Breaking changes
* #1477 WebSharper 10 needs the .NET 10 SDK to be available. All previous .NET core and standard frameworks are still valid as project targets.
* #1528 Within `WebSharper.AspNetCore`, renamed `IWebSharperContentService` to `IWebSharperMvcService`, and `WebSharperContentService` to `WebSharperMvcService`. See [Razor helper docs](https://docs.websharper.com/core/aspnetcore#embed-websharper-controls-into-razor-pages)

## Enhancements
* #1478 Updated F# compiler service used for F# 10 syntax features.
* #1479 Support C# 14.0 features: extension members, null-conditional assignment, field backed properties (the `field` keyword), user-defined compound assignment. 
* #1479 #1531 Linq proxies for `TryGetNonEnumeratedCount`, `SequenceEqual`, `SequenceCompareTo`, `CountBy`, `AggregateBy`, `Index`
* #1476 F# source generators. A new metaprogramming feature that allows generating source files before F# compilation starts. See [F# code generators docs](https://docs.websharper.com/metaprogramming/fsharp-code-generators)
* #1513 Add `changeTracking` setting and output merging. This can be used to make manual edits to WebSharper's JavaScript output and not have them overwritten when those changes can be merged automatically with the new output. See [output merging docs](https://docs.websharper.com/metaprogramming/fsharp-code-generators#output-merging)
* #1526 Allow setting default `RemotingProvider` through `wsconfig.json`
* #1524 #1528 Add more services to `WebSharper.AspNetCore`. See [configuring options and startup docs](https://docs.websharper.com/core/aspnetcore#configuring-options-and-startup)
* #1460 Enhancements to the experimental `.d.ts`/`.ts` output, but it's still exprerimental, there are no full type checking correctness guarantees
* #1538 Add `RequireFeature` attribute and `IBundleExports` (used by UI templating). See [Adding required exports for server-side functionality](https://docs.websharper.com/core/bundling#adding-required-exports-for-server-side-functionality)
* #1537 Automatically include `IInitializer` implementations in production mode sitelet bundles (used by UI templating)
* #1529 Compiler API cleanup. See [compiler API docs](https://docs.websharper.com/metaprogramming/compiler)
* #1555 If a client-side helper is not marked for any bundle in a sitelet project, it's automatically included in all bundles instead of none.

## Fixes
* #1527 Added missing nuget dependencies for `WebSharper.AspNetCore` package
* #1541 Special number names `Infinity` and `NaN` weren't considered properly for shadowing check, breaking if shadowed and numeric value was used
* #1549 `.css` bundles for sitelet pages are linked with proper relative path
* #1551 Add `LogBundleChoice` runtime setting for debugging bundle fallbacks to `all.js`
* #1552 Apply server-side event attributes to possibly multiple elements
* #1553 Fix side-effecting import bad output syntax for `.js` modules and other files like `.css`
* #1554 Fix side-effecting import ordering being changed by inlining optimizations
* #1556 Fix duplicate key error on split proxies in a sitelet project
* #1512 Fixed WIG `=?` operators (should create get-only properties)

## UI Enhancements
* https://github.com/dotnet-websharper/ui/issues/299 Reworked `Flow` API. Now supports tying to routing, stepping back, and resetting/restarting. See [UI flows docs](https://docs.websharper.com/ui/flow)
* https://github.com/dotnet-websharper/ui/issues/307 Added an `.IsEmpty` property for `Doc`, for server side only.

## UI Fixes
* https://github.com/dotnet-websharper/ui/issues/300 fix server-side events and `OnAfterRender` in prebundle sitelet mode (production-ready esbuild mode)
* https://github.com/dotnet-websharper/ui/issues/303 fix server-side `OnAfterRender` with lambda taking no arguments
